# claude-mem — Рекомендация

> Дата: 2026-04-27
> Контекст: agent-skills v4.1, существующая встроенная auto-memory

---

## TL;DR

**Не ставить на основную рабочую машину сейчас.** Перепроверить через 1–2 месяца, когда закроют worker-баги и privacy-leak.

---

## Аргументы против установки

### 1. Дублирование с существующими слоями памяти

У тебя уже два механизма персистентности:

| Слой | Где | Назначение |
|---|---|---|
| Встроенная auto-memory | `~/.claude/projects/.../memory/MEMORY.md` | ручные заметки про user/feedback/project |
| Project memory | `CLAUDE.md` в репо | конвенции, архитектура, версии |

claude-mem добавит **третий слой**, частично перекрывающий первые два. Без чёткой границы — будет путаница, какие данные где искать.

### 2. Критические баги ядра

- Worker регулярно зависает и не генерирует observations
- `<private>` теги текут в LLM-саммаризатор — риск утечки секретов/NDA-кода
- ChromaSearch ранжирует по дате, а не по релевантности — поиск частично сломан

Это не «маленькие issues», а нарушение основной обещанной функциональности.

### 3. Bus factor = 1

823 закрытых PR — все от автора. Если `@thedotmack` потеряет интерес или сменит фокус, поддержка обвалится. Для инструмента, в котором копится твоя память годами, это критичный риск.

### 4. AGPL-3.0

Если форкаешь и публикуешь модифицированную версию — обязан открыть исходники. Для пет-проекта это норма, но если когда-нибудь захочешь интегрировать в коммерческий тулинг — ловушка.

### 5. Bleeding edge — v12.4.7 в 8-месячном проекте

12 мажорных версий за 8 месяцев — значит API нестабилен. Migrate-headache при апдейтах гарантирован.

## Аргументы за установку

### 1. Реальная боль с контекстом существует

Если у тебя длинные сессии и ты часто теряешь нить «что уже исследовано в этом проекте» — claude-mem закроет именно это.

### 2. Послойная выдача — действительно умное решение

`search` → `timeline` → `get_observations` экономит ~10× токенов на больших историях. Встроенная memory грузит весь `MEMORY.md` в контекст каждый раз.

### 3. AST-поиск через tree-sitter

`smart_search` / `smart_outline` — точный поиск символов/функций без полного чтения файла. Это полезная фича и без memory-слоя.

### 4. Активная разработка

Релизы почти ежедневно. Есть шанс, что критические баги закроют быстро.

## Сравнение с SuperClaude (соседнее исследование)

claude-mem и SuperClaude — **разные классы инструментов**, не альтернативы:

| Аспект | claude-mem | SuperClaude |
|---|---|---|
| Класс | Memory-плагин | Meta-фреймворк (prompt-only) |
| Что добавляет | Персистентный контекст между сессиями | 30 команд + 20 агентов + 7 режимов |
| Runtime | TypeScript worker + SQLite + Chroma | Никакого — только .md-файлы |
| Зрелость | Бета, критические баги | Production-ready (v4.3.0) |
| Bus factor | 1 (`@thedotmack`) | Организация SuperClaude-Org |
| Лицензия | AGPL-3.0 + PolyForm | MIT |
| Совместимость | Можно сочетать | Можно сочетать |

**Они не конкурируют** — SuperClaude расширяет «что Claude умеет делать», claude-mem расширяет «что Claude помнит». Технически совместимы.

Для нашего проекта (`agent-skills`) актуальнее SuperClaude — у нас уже своя pipeline-архитектура, и интересны их паттерны (confidence-check, mode inheritance, persona-driven workflows). Из claude-mem стоит подсмотреть только идею послойной выдачи `search → timeline → details`, если будем строить свою memory-систему.

## План если решишь ставить

1. **Не на основную машину** — отдельная виртуалка/контейнер или dev-репо без NDA-кода
2. **Закрой privacy-leak** — пока не закроют issue, не используй с чувствительными данными
3. **Зафиксируй версию** — `npm install claude-mem@12.4.7`, не апдейть автоматически
4. **Бэкапь `~/.claude-mem/`** — SQLite-файл = вся память
5. **Мониторь worker** — `npm run worker:restart` под рукой
6. **Проверь 68K звёзд** — открой страницу репо вручную, агент мог наврать

## План если не ставить (рекомендуемый)

### Что сделать вместо этого

1. **Прокачать встроенную auto-memory.** Сейчас в `MEMORY.md` всего два файла (`feedback_no_speculative_adapters`, `feedback_enforcement_gaps`). Большая часть пользы любой memory-системы — дисциплина записи, а не векторная БД.

2. **Подсмотреть из claude-mem архитектурные идеи без установки:**
   - Послойная выдача `search → timeline → details`
   - Дедупликация observations через content_hash
   - Knowledge Agents — компилируемые «мозги» из истории
   - Worktree adoption при мерже веток

3. **Перечитать через 1–2 месяца.** Дата ревизии: ~2026-06-15.
   - Проверить closed issues на worker и privacy-leak
   - Проверить, появились ли внешние контрибьюторы
   - Проверить, стабилизировался ли API

## Решение

**Вердикт: не ставить.**

Причина одной фразой: *три критических бага в ядре + утечка приватных данных + bus factor = 1 — слишком высокая цена за то, что частично решается дисциплиной записи во встроенную memory.*
